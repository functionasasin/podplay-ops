## Pain Points to Address (from competitive research)

These are the verbatim pain points from Filipino freelancer communities that the landing page must speak to:

1. "I don't know if I should choose 8% or graduated" — regime confusion is the #1 question
2. "I spent ₱8,000 on a CPA for something that's just multiplication"
3. "I chose the wrong option and now I owe more tax and can't change it until next year"
4. "eBIRForms doesn't work on my Mac"
5. "I keep missing the quarterly deadlines (May 15, August 15, November 15)"
6. "My clients withhold tax but I don't know how to account for those 2307s"
7. "I'm employed AND freelancing — which form do I file?"

## Market Data (from market-research aspect, 2026-02-28)

### User Population by Income Tier (for market sizing and messaging)

| Income Band (Annual PHP) | Tax Situation | Estimated PH Freelancers | Tool Value |
|--------------------------|---------------|--------------------------|------------|
| < ₱250,000 | Income tax exempt; still need to file quarterly | ~350,000 | Medium — filing reminders |
| ₱250,000 – ₱500,000 | 8% likely optimal; most don't know this | ~400,000 | High — regime savings reveal |
| ₱500,000 – ₱1,500,000 | All 3 paths meaningfully different | ~500,000 | Very High — core audience |
| ₱1,500,000 – ₱3,000,000 | High expense ratio may favor itemized | ~200,000 | Highest ROI — CPAs charge most |
| > ₱3,000,000 | VAT registered; 8% not available | ~50,000 | High — graduated vs itemized |

### CPA vs Tool Cost Comparison (for landing page "savings" section)
- CPA annual ITR filing only: ₱3,000 – ₱10,000/year
- This tool (pro tier): ₱2,400–₱4,000/year (proposed)
- Savings: ₱600 – ₱8,000/year in CPA fees
- Plus: Regime optimization savings — wrong regime = 20–40% tax overpayment
- Example: ₱1M earner in wrong regime overpays ~₱40,000–₱80,000 in tax per year

### Demographic Profile of Target User
- Age: 18–35 (82% of Asian freelancers under 35)
- Gender: Skewing female (40% of Taxumo users are millennial women)
- Skills: VA (largest), content writing, design, software development, digital marketing
- Income: $300–$1,500/month USD equivalent (₱17K–₱85K/month)
- Primary tax confusion: "Should I choose 8% flat rate?" — asked by virtually every BIR-registering freelancer
- Platforms: Upwork, Fiverr, OnlineJobs.ph, direct clients

### Urgency Drivers (Regulatory Context)
- BIR stepped up enforcement in 2023–2024: Oplan Kandado, bank record subpoenas
- BIR RR 16-2023: E-marketplace withholding (1% on Upwork/Fiverr remittances) — platforms now report to BIR
- EOPT Act (2024): Eliminated ₱500 Annual Registration Fee — lower barrier to formalize
- E-invoicing expanding: Freelancers increasingly visible to BIR
- Filing is now mandatory even for zero-income quarters — no more "too small to care"

### Messaging That Resonates (derived from pain point research)
1. "The BIR now knows about your Upwork earnings" — urgency message for unregistered
2. "8% or Graduated? You shouldn't have to guess." — core value prop for registered-but-confused
3. "Your CPA charges ₱8,000 for a 30-second calculation" — price comparison for existing filers
4. "The wrong choice costs you ₱40,000+ per year and you can't undo it until next April" — irrevocability urgency
5. "Works on Mac, Windows, mobile — unlike eBIRForms" — platform jab
6. "You're paying for your CPA's time, not expertise" — repositioning what CPA ITR filing is

## Interactive Elements Specification

### Embedded Tool (Hero CTA)

When the user clicks "Compute My Tax Now — Free" or any primary CTA on the landing page, they are navigated to `/compute` — the full wizard page. The landing page does NOT embed the wizard inline in the page.

**Rationale:** Inline embedding would require the full wizard to load in the landing page (heavy), complicate A/B testing, and reduce clarity of the hero message. Dedicated `/compute` route gives the wizard full page real-estate and a dedicated URL for SEO.

**Exception — Demo Section:** The demo section (`#demo`) shows a static screenshot, not an interactive tool. Users who want to interact must click the "Compute Your Tax — It's Free" CTA below the screenshot.

### Scroll-Triggered Elements

| Trigger | Element | Animation |
|---------|---------|-----------|
| Stats strip visible (IntersectionObserver) | Stat numbers | Count-up animation from 0 to final value over 1,200ms with easeOutCubic |
| Problem blocks visible | Each card | Fade-in + slight upward translate (0→1 opacity, translateY(16px→0), staggered 100ms per card) |
| How It Works steps visible | Step circles | Scale from 0.8→1.0 + fade-in, staggered 150ms per step |
| Regime cards visible | Cards | Fade-in left→right stagger, 100ms each |
| Testimonial cards visible | Cards | Fade-in, staggered 100ms per card |

All animations must respect `prefers-reduced-motion: reduce` (disable all animations, show elements at final state immediately).

### Sticky Floating CTA (Mobile)

On mobile devices (≤768px), after scrolling past the hero section (>100vh), display a sticky bottom bar:

- Height: 64px
- Background: white, `box-shadow: 0 -2px 8px rgba(0,0,0,0.10)`
- Contents: "Compute Your Tax" text (Font Inter 600 15px, `color-text-primary`) + "Free →" button (primary-solid, 44px height, compact padding, border-radius 8px).
- CTA link: `/compute`
- Hidden when hero is visible. Hidden on desktop.

## Conversion Optimization Specifications

### Page Load Performance Requirements

| Metric | Target | Rationale |
|--------|--------|-----------|
| Core Web Vitals LCP | < 2.5 seconds | Google ranking factor; users bounce at >3s |
| Core Web Vitals FID/INP | < 100ms | Interactive feel |
| Core Web Vitals CLS | < 0.1 | No layout shift during load |
| Time to First Byte | < 800ms | Vercel Edge Network |
| Lighthouse Performance Score | ≥ 90 | Aggregate performance |
| Page weight (initial JS + CSS) | < 200KB gzipped | Exclude hero illustration (lazy-loaded) |

**Implementation:** The landing page is a Next.js static page (SSG). Hero illustration is loaded with `loading="eager"` and `fetchpriority="high"`. Demo screenshot and other below-fold images use `loading="lazy"`. All fonts loaded via `next/font` for zero layout shift.

### Conversion Funnel Tracking

Every section view and CTA click must be tracked in PostHog:

| Event | Properties |
|-------|-----------|
| `page_view` `{page: 'landing', referrer, utm_*}` | Fires on mount |
| `section_viewed` `{section_id}` | Fires when IntersectionObserver triggers at 50% visibility |
| `cta_clicked` `{cta_id, section_id, variant}` | Fires on any CTA click |
| `nav_link_clicked` `{link_text, destination}` | Fires on nav link click |
| `faq_opened` `{question_index}` | Fires when FAQ accordion opens |
| `pricing_toggle` `{selected_billing_cycle}` | Fires on Monthly/Annual toggle |
| `tool_started` | Fires when user navigates to `/compute` from landing page |

### Exit Intent

On desktop, when the cursor moves above the browser viewport boundary (intending to close tab), show an exit-intent modal:

- Trigger: `mouseleave` on `document` when `clientY < 0`.
- Modal: centered overlay, blur backdrop.
- Heading: "Before you go — see if you're overpaying"
- Body: "Over 60% of Filipino freelancers are using the wrong tax regime. It takes 30 seconds to check."
- CTA: "Compute My Tax — It's Free" → `/compute`.
- Dismiss: "No thanks" text link closes modal and sets cookie `exit_intent_dismissed` (90-day TTL). Do not re-trigger after dismissal.
- Mobile: No exit-intent on mobile (touch devices have no cursor).

### Page Speed Optimization (Specific Implementation)

1. Hero illustration: served as WebP/AVIF with PNG fallback. Dimensions: 480×400px at 1× (960×800px for 2× Retina). Compressed to ≤80KB at 1×.
2. Demo screenshot: served as WebP/AVIF. 880×560px at 1×. Compressed to ≤120KB. Lazy-loaded.
3. Testimonial section: no external images (text-only testimonials with CSS avatar initials).
4. No external font requests (Inter loaded via next/font/google, self-hosted).
5. No external script dependencies on the landing page except: PostHog (analytics, async), Google Tag Manager (deferred).
