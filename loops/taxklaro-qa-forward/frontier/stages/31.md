# Stage 31 — Discovery: Route-Component Wiring

## Instructions
1. Open `router.ts` — list every route
2. For each route, open the route file and verify:
   - It imports and renders at least one component from `src/components/`
   - It's NOT a stub (`return null` or `return <div>placeholder</div>`)
   - The parent route is correct (authenticated pages under `authenticatedRoute`, public under `publicRootRoute`)
3. Check for orphaned components in `src/components/` that are never imported
4. Fix any issues found
5. Build + test

## Pass criteria
- Every route renders real content
- No stubs
- No orphaned components (or note them for removal)
