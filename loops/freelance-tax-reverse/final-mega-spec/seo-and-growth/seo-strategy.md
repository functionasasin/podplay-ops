### 6.7 Core Web Vitals Targets

All pages must meet these performance benchmarks (measured by Google PageSpeed Insights, lab conditions):

| Metric | Target (Mobile) | Target (Desktop) | Measurement Tool |
|--------|----------------|-----------------|-----------------|
| Largest Contentful Paint (LCP) | ≤ 2.5 seconds | ≤ 1.5 seconds | PageSpeed Insights, CrUX |
| Interaction to Next Paint (INP) | ≤ 200 milliseconds | ≤ 100 milliseconds | PageSpeed Insights, CrUX |
| Cumulative Layout Shift (CLS) | ≤ 0.1 | ≤ 0.05 | PageSpeed Insights, CrUX |
| First Contentful Paint (FCP) | ≤ 1.8 seconds | ≤ 1.0 second | PageSpeed Insights |
| Total Blocking Time (TBT) | ≤ 200 ms | ≤ 150 ms | PageSpeed Insights |
| Time to First Byte (TTFB) | ≤ 800 ms | ≤ 400 ms | PageSpeed Insights |
| PageSpeed Score (Mobile) | ≥ 85 | ≥ 90 | PageSpeed Insights |

Implementation requirements to meet these targets:
1. All tool computation is client-side JavaScript; no server round-trip required for computation
2. Hero image (landing page) must be lazy-loaded below the fold; no LCP image above fold unless preloaded
3. Fonts (Inter): use `font-display: swap` and preload the regular and bold variants only
4. No layout shift from ads (no ads at launch)
5. Computed results section uses skeleton loaders, not blank-to-content jumps
6. All images: WebP format, `width` and `height` attributes specified in HTML to prevent CLS
7. Vercel Edge Network CDN ensures TTFB ≤ 200ms from Singapore edge to Philippine users

## 7. On-Page SEO Specifications

### 7.1 Heading Hierarchy Rules

Every page must follow this heading structure:

| Element | Rule |
|---------|------|
| `<h1>` | Exactly one per page. Contains the primary keyword for that page. Written as a user-facing headline, not a keyword stuffed phrase. |
| `<h2>` | Major sections. Should include secondary keywords naturally. Maximum 6 per page (excluding blog posts). |
| `<h3>` | Subsections under `<h2>`. Can be used freely within blog posts. |
| `<h4>` | Minor subsections. Use sparingly — prefer `<strong>` for inline emphasis. |

H1 examples per page:
- Homepage: `"Which Tax Regime Saves You the Most? Compute All Three in 30 Seconds."` (targets "income tax calculator philippines freelancer")
- Compare 8% vs Graduated: `"8% Flat Rate vs. Graduated Income Tax Philippines: Side-by-Side Comparison"` (targets K-007)
- Pillar guide: `"Philippine Freelancer Income Tax: The Complete 2026 Guide"` (targets K-009/K-010)
- Blog post on 8% option: `"8% Income Tax Option Philippines: Complete Guide for Freelancers (2026)"` (targets K-011)

### 7.2 Image Alt Text Requirements

Every `<img>` on a public page must have a descriptive `alt` attribute.

| Context | Alt Text Pattern | Example |
|---------|----------------|---------|
| Regime comparison results screenshot | `"TaxKlaro showing [REGIME] as optimal for ₱[AMOUNT] gross receipts"` | `"TaxKlaro showing 8% flat rate as optimal for ₱800,000 gross receipts"` |
| BIR form screenshot | `"BIR Form [FORM-NUMBER] — [section description]"` | `"BIR Form 1701Q Schedule II — Q1 cumulative income tax computation"` |
| Diagram/chart | Describe what the chart shows | `"Bar chart comparing 8% flat rate, OSD, and itemized deductions tax at ₱1 million gross receipts"` |
| Logo (inline) | `"TaxKlaro logo"` | `"TaxKlaro logo"` |
| Decorative image with no content meaning | `alt=""` (empty, not missing) | `alt=""` |
| Blog post hero image | Describe the image content and relation to post | `"Filipino freelancer working on laptop with BIR tax documents"` |

### 7.3 Internal Linking Strategy

Internal links transfer page authority and improve crawl efficiency.

**Required internal links per page:**

| Page | Must Link To |
|------|-------------|
| Homepage | Compare pages (8% vs Graduated, OSD vs Itemized), Pillar guide, Pricing, Blog index |

| Compare: OSD vs Itemized | Homepage (tool), Blog: OSD vs itemized post, Blog: itemized deductions post, Pillar guide |

**Anchor text rules:**
- Never use "click here" or "learn more" alone as anchor text
- Use descriptive anchors: "compute your tax under all three regimes" (links to /), "8% vs Graduated comparison" (links to /compare/8-percent-vs-graduated)
- Vary anchor text for the same destination — do not use identical anchor text every time
- Maximum 1 exact-match keyword anchor per page for each destination

## 10. Local and Philippines-Specific SEO Signals

### 10.1 Country Targeting Signal

In Google Search Console, set geographic target to **Philippines** for the `taxklaro.ph` property. The .ph TLD is itself a strong geographic signal; the GSC setting reinforces it.

### 10.2 Philippine Tax Year References

All blog posts and the pillar guide must include the current tax year in the title (e.g., "2026") and update it annually. Search intent for "freelance tax philippines" peaks in March–April and is heavily year-specific.

Update schedule:
- Pillar guide `<title>` and H1: Update to new year by January 15 each year
- All blog post slugs: Do NOT include year in slug (to avoid URL changes on annual update); include year in title tag and H1 only
- Blog posts on filing calendar and deadlines: Update deadline dates annually, update `dateModified` in schema

### 10.3 Filipino Language Signals

The site is in English (en-PH). Incorporate Filipino/Tagalog tax terminology naturally within English content to capture mixed-language searches:

| Filipino Term | When to Include |
|-------------|----------------|
| "income tax" | Standard; use as-is |
| "pagbabayad ng buwis" | Include once in pillar guide intro paragraph |
| "BIR" (Bureau of Internal Revenue) | Always use BIR abbreviation; Filipino users search "BIR" not "Bureau of Internal Revenue" |
| "Bir form" (lowercase) | Include in FAQ schema (users often don't capitalize) |
| "1701q" vs "Form 1701Q" | Include both variants in heading H2s and body text |

## 11. Competitor SEO Notes (Updated)

- **Taxumo** dominates informational keywords but their tool pages require signup + ₱2,499/quarter minimum. Free tool pages will outrank subscription-gated pages on transactional queries.
- **JuanTax** strong on accountant-focused queries ("BIR eTSP accredited") — do not compete directly on those; focus on freelancer-specific queries.
- **codingace.net** ranks for some 8% calculator queries but has zero brand authority, no content, no HTTPS certificate consistently — an easy target.
- **No competitor** has a regime comparison comparison page — this remains an uncontested SERP position at launch.
- **Seasonal SEO**: April (annual ITR) generates 3× normal traffic. Content published in February–March will be most effective for the April filing season.
- **BIR PDF guides**: The BIR website ranks for many exact-match queries (e.g., "Form 1701A instructions") — do not try to outrank BIR.gov.ph for its own form documentation pages. Instead, target "how to" and "which to use" queries around those forms.
