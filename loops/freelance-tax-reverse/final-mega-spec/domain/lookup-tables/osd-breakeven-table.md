## Table 2: OSD vs. 8% Flat Rate Comparison (for GR ≤ ₱3,000,000, Pure Business)

For taxpayers eligible for the 8% option (gross ≤ ₱3M, not VAT-registered):

| Gross Receipts | Path B Total (OSD) | Path C Total (8%) | Savings from 8% | 8% Effective Rate |
|---------------|-------------------|------------------|-----------------|-------------------|
| ₱100,000 | ₱3,000 | ₱0 | ₱3,000 | 0.00% |
| ₱200,000 | ₱6,000 | ₱0 | ₱6,000 | 0.00% |
| ₱250,000 | ₱7,500 | ₱0 | ₱7,500 | 0.00% |
| ₱300,000 | ₱9,000 | ₱4,000 | ₱5,000 | 1.33% |
| ₱400,000 | ₱12,000 | ₱12,000 | ₱0 | 3.00% |
| ₱416,667 | ₱12,500 | ₱13,333 | -₱833 | 3.20% |

Wait — at GR = ₱400,000:
- Path C: (400,000 − 250,000) × 0.08 = 150,000 × 0.08 = ₱12,000
- Path B: graduated(240,000) + 12,000 = 0 + 12,000 = ₱12,000 → TIED

At GR = ₱416,667:
- Path C: (416,667 − 250,000) × 0.08 = 166,667 × 0.08 = ₱13,333
- Path B: graduated(250,000) + 12,500 = 0 + 12,500 = ₱12,500 → OSD WINS by ₱833

This is the ONE crossover: at low gross receipts just above ₱400,000, OSD can edge out 8% slightly. Let me compute the full crossover range more carefully.

OSD: total = graduated(0.60 × GR) + 0.03 × GR
8%: total = (GR − 250,000) × 0.08

For GR between ₱400,001 and ₱416,666:
- OSD: graduated(0.60×GR) = 0 (net < 250,000), so total = 0.03×GR
- 8%: total = (GR − 250,000) × 0.08
- OSD wins when: 0.03×GR < (GR − 250,000) × 0.08
  → 0.03×GR < 0.08×GR − 20,000
  → 20,000 < 0.05×GR
  → GR > 400,000

So OSD beats 8% ONLY in the range ₱400,001 to ₱416,666 (when OSD net < ₱250,000 but 8% applies without the zero-bracket having full effect).

At GR = ₱416,667: OSD net = 250,000 (0% IT bracket ceiling), 8% base = 166,667 × 0.08 = 13,333. OSD total = 12,500. OSD still wins marginally.

At GR = ₱416,668: OSD net = 250,001; IT = 0.15; OSD total = 0.15 + 12,500.04 = 12,500.19; 8% total = 13,333.44. OSD still wins.

The crossover where 8% definitively beats OSD again:
For GR in bracket 2 OSD range (416,667 < GR ≤ 666,667 where OSD net is 250,001–400,000):
- OSD total = (0.60×GR − 250,000)×0.15 + 0.03×GR
- 8% total = (GR − 250,000)×0.08

Set equal:
(0.60×GR − 250,000)×0.15 + 0.03×GR = (GR − 250,000)×0.08
0.09×GR − 37,500 + 0.03×GR = 0.08×GR − 20,000
0.12×GR − 37,500 = 0.08×GR − 20,000
0.04×GR = 17,500
GR = 437,500

So:
- At GR < ₱400,000: OSD beats 8% (8% has no advantage below the OSD net 250K boundary)
- At GR = ₱400,000 to ₱437,500: OSD beats 8% (tiny window; max OSD advantage ~₱833 at ~₱416,667)
- At GR > ₱437,500: 8% beats OSD consistently and by increasing margin

## Table 3: OSD Breakeven vs. Itemized Deductions

For all taxpayers — the breakeven is definitional (40% of gross receipts):

| Gross Receipts | OSD (40%) | Breakeven Expense (40%) | If AE < 40%: Winner | If AE > 40%: Winner |
|---------------|-----------|------------------------|---------------------|---------------------|
| ₱500,000 | ₱200,000 | ₱200,000 | OSD | Itemized |
| ₱1,000,000 | ₱400,000 | ₱400,000 | OSD | Itemized |
| ₱1,500,000 | ₱600,000 | ₱600,000 | OSD | Itemized |
| ₱2,000,000 | ₱800,000 | ₱800,000 | OSD | Itemized |
| ₱3,000,000 | ₱1,200,000 | ₱1,200,000 | OSD | Itemized |
| ₱5,000,000 | ₱2,000,000 | ₱2,000,000 | OSD | Itemized |

Rule: OSD breakeven always exactly at 40% expense ratio. No income level affects this breakeven because:
- OSD deduction = 40% × GR
- When actual expenses (AE) = 40% × GR → both paths produce identical NTI → identical tax
- The graduated rate table is irrelevant to this breakeven (same rate applies to same NTI)

## Table 4: OSD for Mixed Income Earners

Mixed income earner: Has both compensation income AND business/professional income.
OSD applies ONLY to business portion. Compensation has no deduction.

The total NTI and income tax:
- Total_NTI = compensation + (biz_gross_receipts × 0.60)
- IT = graduated_tax(Total_NTI)
- Less: employer WHT on compensation + CWT credits + quarterly advance payments

Example — Employee with ₱600,000 annual salary + ₱500,000 freelance income (OSD):
- Compensation NTI: ₱600,000
- Business NTI (OSD): ₱500,000 × 0.60 = ₱300,000
- Total NTI: ₱900,000
- IT: 102,500 + (900,000−800,000)×0.25 = 102,500+25,000 = ₱127,500
- Less: Employer WHT on ₱600,000 salary: using graduated = 102,500 + (600,000−800,000)×0.25... No wait. The employer withholds on the ₱600,000 compensation only: graduated(600,000) = 22,500 + (600,000−400,000)×0.20 = 62,500. Actually: 102,500 + (600,000-800,000)×... 600,000 < 800,000 so bracket 3: 22,500 + (600,000-400,000)×0.20 = 22,500+40,000 = ₱62,500 employer WHT.
- Balance payable at annual filing: 127,500 − 62,500 = ₱65,000 (before CWT credit)
- Note: Mixed income earner CANNOT use 8% and OSD simultaneously for same income. Must choose one method for business income for the year.

## Table 5: OSD for Traders (Gross Income Base)

For businesses that sell goods (not services), OSD applies to gross income = gross sales − COGS.

| Gross Sales | COGS | Gross Income (OSD Base) | OSD (40% of GI) | Net Taxable Income | Notes |
|------------|------|------------------------|----------------|-------------------|-------|
| ₱1,000,000 | ₱300,000 | ₱700,000 | ₱280,000 | ₱420,000 | High-margin trader |

Key distinction for traders:
- OSD base = Gross Income (after COGS), NOT gross sales
- "Operating expenses" (rent, salaries, utilities) are NOT separately deducted — they're included in the 40% OSD
- A trader with ₱1M sales, ₱500K COGS, ₱250K operating expenses: under OSD gets ₱200K deduction vs ₱250K actual OpEx → Itemized wins (actual OpEx > 40% of GI = ₱200K)

## Table 6: OSD Election Decision Matrix

Used by the engine to determine whether the user's OSD election is valid:

| Condition | Result |
|-----------|--------|
| First quarterly return already filed with itemized expenses | OSD NOT available for this year |
| First quarterly return filed with no deduction method selected | Default: itemized (OSD requires explicit election) |
| First quarterly return filed, checkbox "OSD" ticked | OSD elected, irrevocable for year |
| New registrant; no quarterly return filed yet | OSD available; must elect in first quarterly return |
| Current quarter is Q2, Q3, or Q4; Q1 already filed without OSD election | OSD NOT available; locked into itemized |
| User was on 8% in Q1 and now wants OSD | NOT possible; must remain on 8% for year or drop to graduated + itemized |
| User elected OSD in Q1 but actual expenses > 40% | Stuck on OSD; cannot switch; must note in output |
| Prior year was OSD; no action taken in Q1 of current year | OSD does NOT carry over; must re-elect; if Q1 not re-elected → itemized |

## Rounding Rules for OSD Computation

1. Compute OSD amount: `round(gross_receipts × 0.40, 2)` (to centavo)
2. Compute NTI: `gross_receipts − osd_amount` (may not equal exactly 60% due to rounding; use subtraction, not multiplication)
3. Apply graduated tax: uses NTI computed in step 2
4. Percentage tax: `round(gross_receipts × 0.03, 2)`
5. Total: sum of income tax + percentage tax (no further rounding)

Example (GR = ₱500,000.01):
- OSD = round(500,000.01 × 0.40, 2) = round(200,000.004, 2) = ₱200,000.00
- NTI = 500,000.01 − 200,000.00 = ₱300,000.01
- IT = (300,000.01 − 250,000) × 0.15 = 50,000.01 × 0.15 = round(7,500.0015, 2) = ₱7,500.00
- PT = round(500,000.01 × 0.03, 2) = round(15,000.0003, 2) = ₱15,000.00
- Total = ₱22,500.00
