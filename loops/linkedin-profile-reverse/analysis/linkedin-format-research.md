# LinkedIn Format Research

## Character Limits — Complete Reference

| Section | Limit | Notes |
|---|---|---|
| Headline (desktop) | 220 chars | Also appears in search results, connection requests, every comment |
| Headline (mobile) | 240 chars | Slightly more forgiving |
| About / Summary | 2,600 chars | ~300-350 words optimal range: 1,800–2,200 chars |
| Experience Title | 100 chars | Industry-standard titles improve search visibility |
| Experience Description | 2,000 chars | Per entry |
| Publication Title | 250 chars | |

## Truncation Behavior — Critical for Hooks

### About Section
- **Desktop**: First ~300 characters are visible before "See more"
- **Mobile**: Even less — approximately 140–200 characters
- **Rule**: The first 2-3 lines (300 chars) must stand alone as a hook. If someone only reads those and clicks away, they should still walk away with a clear impression.

### Posts
- First ~140 characters visible on mobile
- First ~200–210 characters visible on desktop before "See more"
- Over 60% of LinkedIn usage is on mobile — design for mobile-first

### Headline
- Full headline appears in search results and profile card
- For maximum impact, put the primary keywords + value prop in the **first 100 characters** (the rest may get cut in narrow display contexts)

## Profile Sections — Structure & Ordering

### How LinkedIn profiles render (top to bottom):
1. **Photo + Banner image** — first visual impression
2. **Name + Headline** — most-viewed text
3. **Location + Contact info + custom URL button**
4. **Featured section** — VISUAL, thumbnail-driven, prime real estate
5. **About/Summary** — truncated at ~300 chars on desktop
6. **Activity** — recent posts (if Creator Mode on)
7. **Experience** — job entries, most recent first
8. **Education**
9. **Skills & Endorsements** — top 3 most visible
10. **Recommendations**
11. **Publications, Certifications, etc.**

### Featured Section Details
- Appears **above** Activity and Experience — very high placement
- Supports: LinkedIn posts, LinkedIn articles, external links, documents (PDF, images, up to 100MB)
- Shows **thumbnail preview** — visual content outperforms plain links
- LinkedIn auto-generates thumbnail from URL; can also upload custom image
- Recommended: 4–6 items, curated, ordered by importance (drag to reorder)
- First item in Featured is the "hero" — use it for the single most impressive thing

## LinkedIn Search / SEO Mechanics

### Keyword Weight (highest to lowest):
1. **Headline** — single most important field for search ranking
2. **About section** — 8–12 relevant keywords naturally integrated
3. **Experience titles** — use industry-standard titles (searchable terms)
4. **Skills** — top skills with endorsements rank higher
5. **Profile completeness** — complete profiles get 30% more views

### Practical implications:
- Don't use invented titles that no one searches for. "Polymath Builder" is cool in an About section but terrible for search — also include terms like "Software Engineer," "AI Engineer," "ML Engineer," "Founding Engineer," "CTO"
- Skills list is a keyword bank — add all relevant technical terms
- Headline real estate: use pipes or bullets to pack in more keywords while maintaining readability
  - Example: `AI Agent Infrastructure | Probabilistic ML | Founder | Southeast Asia`

### Creator Mode
- Enables: Follow button (instead of Connect), newsletter, link in bio
- Shows Activity section higher on profile
- Useful if actively posting content; not required for builder archetype

## Mobile Rendering — Key Constraints

- 60%+ of LinkedIn usage is mobile
- Headline is the most visible element — show on every search result card
- About section: design for 140-char cold hook, 300-char warm hook, full 2600-char payload
- Experience entries: bullet points render as full paragraphs on some mobile views — use bold text sparingly, avoid tables/special formatting
- Featured section: thumbnails show clearly on mobile — use high-contrast images

---

## What Gets Hidden / Truncated

- Experience descriptions: Only first 3-4 lines visible before "See more" (per entry)
- Skills: Only top 3 shown by default; rest behind "Show all X skills"
- Recommendations: Collapsed by default
- Featured section: Scrollable carousel on mobile — first 1-2 items get the most views

---

## Formatting Tricks

- **Emoji as bullet points**: Acceptable and increasingly common for technical profiles; use sparingly (1-2 emojis max in headline)
- **Line breaks in About**: LinkedIn collapses excessive whitespace but allows paragraph breaks; use blank lines between sections
- **Bold text**: Not natively supported in profile text fields — some people use Unicode bold characters (𝗕𝗼𝗹𝗱) but this looks hacky on some systems
- **Special characters**: Pipes (|), em dashes (—), arrows (→) work well in headlines for visual separation

## The "First Impression" Window

What a visitor sees before doing anything:
1. Photo + banner
2. Name
3. Full headline
4. "Follow" / "Connect" button
5. Location
6. Featured section (top item, thumbnail)
7. First 300 chars of About section

This is the entire decision window for whether they read further. Every one of these elements must work in concert.

## Custom URL Recommendation

- Default: `linkedin.com/in/random-string`
- Target: `linkedin.com/in/clsandoval` (if available)
- Max 29 chars for the path segment
- Use real name — easier to remember and share

---

## Execution Order for Profile Update

1. Set custom URL first (affects how link is shared)
2. Update headline (immediate SEO impact)
3. Update About section (hook lines crucial)
4. Update/add Featured section items
5. Update Experience entries (most recent first)
6. Update Skills list (reorder top 3 to most impressive)
7. Add Publications section (IEEE TENCON 2022 paper)
8. Upload banner image if desired
