## Tone Guide

### Voice: Builder Confidence + Tinkerer Energy

**NOT**: "Experienced software engineer with expertise in machine learning, computer vision, and autonomous systems development."

**YES**: "i build agents that build things. this readme was drafted by a ralph loop."

**NOT**: "I am passionate about the intersection of artificial intelligence and real-world systems."

**YES**: "the monorepo is a personal OS. my swim coach is a computer. my research agents run on cron."

### Rules

1. **Specific over vague** — "420 commits on a Discord-based org OS at pymc-labs" > "production AI experience"
2. **Numbers as confidence signals** — 1,034 entities, 2,652-line spec, 1,000+ commits, 30-minute CI cadence
3. **The meta-moment is earned** — Saying "this profile was generated by a ralph loop" is not cringe if the loop actually exists (it does). Own it.
4. **Short sentences** — The README should read fast. No paragraphs. Bullets, short lines, whitespace.
5. **No badges or widgets** — GitHub stats widgets, visitor counts, language pie charts are resume-optimization cringe. Text-only, personality-first.
6. **Don't explain everything** — Surface enough to make someone curious, then point to the repos/monorepo. A good README creates pull, not push.
7. **Lowercase personality** — "i build agents that build things" has a different energy than "I build agents that build things." Choose based on overall tone.
