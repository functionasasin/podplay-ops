# Stage 033 — Toast System

## Spec Files
- `ui-spec/toast-messages.md`

## Instructions
1. Install `sonner` (or use shadcn toast if already available). Add `<Toaster />` to the app root.
2. Create `src/lib/toast-messages.ts` with all 65 toast operation constants from the spec. Each constant defines: `message` (string), `duration` (ms), `type` ('success' | 'error' | 'info' | 'warning').
3. Create `src/lib/toast.ts` with a wrapper function: `showToast(operation: string)` that looks up the operation key in toast-messages, then fires the toast with the correct message, duration, and type.
4. Export individual typed helpers: `toastSuccess(operation)`, `toastError(operation)` for convenience.
5. Every message, duration, and type must match `ui-spec/toast-messages.md` exactly.

## Verify
- `npm run build` — build passes with zero errors
