# Stage 151 — Keyboard Navigation

## Spec Files
- `ui-spec/keyboard-nav.md`

## Instructions
1. Add a skip-to-content link as the first focusable element in `AppLayout`.
2. Define ARIA landmark regions: `<nav>` for sidebar, `<main>` for content, `<header>` for top bar.
3. Ensure all interactive elements have logical tab order (no positive tabIndex values).
4. Ensure Enter key submits all forms.
5. Ensure Escape key closes all modals, dialogs, sheets, and dropdowns.
6. Add focus trap to all modal/dialog components (focus stays within while open).
7. Return focus to trigger element when modal closes.
