# Stage 014 — Inventory Tables: inventory, inventory_movements

## Spec Files
- `data-model/schema.md`
- `data-model/relationships.md`

## Instructions
1. Create `apps/podplay/supabase/migrations/00004_inventory_tables.sql`
2. Define `inventory` table:
   - id (uuid PK), item_id (FK to hardware_catalog, unique), quantity_on_hand (integer), quantity_allocated (integer), reorder_point (integer), updated_at
3. Define `inventory_movements` table:
   - id (uuid PK), item_id (FK to hardware_catalog), movement_type (enum: received, allocated, shipped, adjusted, returned), quantity (integer, can be negative for adjustments), reference_type (nullable — e.g., 'project', 'purchase_order'), reference_id (nullable uuid), notes, created_at
4. Add FK constraints: inventory.item_id references hardware_catalog(id)
5. Add indexes: inventory(item_id), inventory_movements(item_id), inventory_movements(created_at)
6. Enable RLS with authenticated read/write policies

## Verify
