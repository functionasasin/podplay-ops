# Stage 010 — Core Tables: projects, installers, settings

## Spec Files
- `data-model/schema.md` (tables section)
- `data-model/relationships.md`

## Instructions
1. Create `apps/podplay/supabase/migrations/00002_core_tables.sql`
2. Define `projects` table with all columns from spec:
   - Identity: id (uuid PK), project_number, name
   - Customer fields: customer_name, customer_email, customer_phone, customer_address
   - Venue fields: venue_name, venue_address, venue_type
   - Config: service_tier, zone_count, speaker_count
   - ISP: isp_provider, isp_speed, isp_status
   - Deployment: deployment_phase, deployment_status, install_date
   - Lifecycle: project_status, created_at, updated_at
3. Define `installers` table: id, name, email, phone, rate, status, created_at
4. Define `settings` table: id, key (unique), value (jsonb), updated_at
5. Add RLS policies: authenticated users can read/write all rows (single-tenant)
6. Add indexes: projects(project_status), projects(deployment_phase), projects(service_tier)
