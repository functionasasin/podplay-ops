# Stage 061 — Intake Submit Logic

## Spec Files
- `business-logic/customer-onboarding.md`
- `business-logic/bom-generation.md`

## Instructions
1. Implement the submit handler in the intake route (`src/routes/projects/$projectId/intake.tsx`).
2. On submit: update the project row in Supabase with all form data from the 7-step wizard.
3. Call the BOM generation function (from stage 063) with the relevant inputs: `service_tier`, `court_count`, `door_count`, `camera_count`, `has_front_desk`, `has_pingpod_wifi`.
4. Run the cost chain calculation for each BOM item and store the results.
5. Update the project `status` to `'procurement'`.
6. Show a success toast on completion and navigate to the project detail page.
7. Show an error toast if any step fails, with the error message.
