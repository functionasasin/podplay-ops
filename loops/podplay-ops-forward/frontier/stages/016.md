# Stage 016 — Financial Tables: invoices, expenses, cc_terminals, replay_signs

## Spec Files
- `data-model/schema.md`
- `data-model/relationships.md`

## Instructions
1. Create `apps/podplay/supabase/migrations/00005_financial_tables.sql`
2. Define `invoices` table:
   - id (uuid PK), project_id (FK to projects), invoice_number, type (enum: deposit, final, change_order), amount (numeric), tax_amount (numeric), total_amount (numeric), status (invoice_status enum), payment_method (payment_method enum, nullable), issued_date, due_date, paid_date (nullable), notes, created_at, updated_at
3. Define `expenses` table:
   - id (uuid PK), project_id (FK to projects, nullable — some expenses are general), category (expense_category enum), description, amount (numeric), payment_method (payment_method enum), vendor, receipt_url (nullable), expense_date, notes, created_at
4. Define `cc_terminals` table:
   - id (uuid PK), project_id (FK to projects), serial_number, model, status (enum: ordered, received, configured, deployed, returned), deployed_date (nullable), notes, created_at, updated_at
5. Define `replay_signs` table:
   - id (uuid PK), project_id (FK to projects), quantity (integer), status (enum: ordered, produced, shipped, installed), order_date, ship_date (nullable), install_date (nullable), notes, created_at, updated_at
6. Add FK constraints (CASCADE on project delete for all)
7. Add indexes: invoices(project_id), invoices(status), expenses(project_id), expenses(category)
8. Enable RLS with authenticated read/write policies

## Verify
- `cd apps/podplay && npx supabase db reset` — exits 0, all financial tables created
