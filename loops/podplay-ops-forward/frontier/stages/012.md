# Stage 012 — Hardware Tables: hardware_catalog, bom_templates, project_bom_items

## Spec Files
- `data-model/schema.md`
- `data-model/relationships.md`

## Instructions
1. Create `apps/podplay/supabase/migrations/00003_hardware_tables.sql`
2. Define `hardware_catalog` table:
   - id (uuid PK), sku (unique), name, vendor, category, unit_cost (numeric), description, is_active, created_at, updated_at
3. Define `bom_templates` table:
   - id (uuid PK), tier (service_tier enum), item_id (FK to hardware_catalog), default_quantity, quantity_rule (jsonb — rules for scaling by zone/speaker count), is_required, created_at
4. Define `project_bom_items` table:
   - id (uuid PK), project_id (FK to projects), catalog_item_id (FK to hardware_catalog), quantity, unit_cost_override (nullable numeric), notes, created_at, updated_at
5. Add FK constraints with appropriate ON DELETE behavior (RESTRICT for catalog, CASCADE for project)
6. Add indexes: hardware_catalog(sku), hardware_catalog(category), project_bom_items(project_id)
7. Enable RLS with authenticated read/write policies
