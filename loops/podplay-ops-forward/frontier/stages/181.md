# Stage 181 — Schema Mismatch Audit

## Spec Files
- `data-model/schema.md`
- `data-model/migration.sql`
- `data-model/relationships.md`

## Instructions
1. Read all Supabase migration files. List every table and column defined in the schema.
2. Grep `src/` for all Supabase query patterns: `.from('`, `.select('`, `.insert(`, `.update(`, `.delete(`.
3. Cross-reference every table/column referenced in code against the migration schema.
4. Fix any mismatches: if code references a column that does not exist, either add a migration or fix the code to match the schema.
5. If the schema has tables/columns not referenced in code, note them but do not remove (they may be used by RLS or triggers).
6. Run build and tests to confirm.
