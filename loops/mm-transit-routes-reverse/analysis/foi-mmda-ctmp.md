## Key Insight for Future Iterations

The Philippine eFOI system (foi.gov.ph) has three structural limitations for this loop:

1. **Programmatic inaccessibility**: HTTP 403 blocks all automated access — the portal is effectively browser-only
2. **Email-only delivery**: Successful FOI requests deliver via email, not public URLs — no searchable document cache
3. **Denial risk**: MMDA actively denies some transit operational data requests (EDSA Carousel data was denied)

**Implication**: The FOI channel is viable only if a human files, waits for approval, and manually reviews the emailed document. This is outside the scope of an automated loop iteration.

**Recommendation for Wave 3**: Do not use FOI portal as a data source. Rely on OSM, Sakay.ph GTFS, TUMI Datahub, and news-sourced stop lists for GTFS synthesis.
