## Overview

This file specifies how SSR consumer panel results render in Discord. Discord has a hard 2000-character message limit per message. The bot uses threads for long-form output. The w2-tool-panel-results analysis established the output format sizing and rendering constraints documented here. The w3-discord-ux aspect will fill in the full interactive UX flow (progress indicators, panel creation acknowledgment, run progress, etc.).

## 1. Message Length Estimates by Format

The following character counts are estimates based on the example outputs in `panel-results.md` and `panel-run.md`. They guide the bot's decision of whether to post inline or use a thread.

### Single-Run Retrieval

| Format | Dimensions | Panel Size | Estimated Output Length | Discord Delivery |
|--------|-----------|-----------|------------------------|-----------------|
| `summary` | 1 | any | ~350–500 chars | Single message (no thread needed) |
| `summary` | 3 | any | ~900–1,400 chars | Single message |
| `summary` | 5 | any | ~1,500–2,100 chars | **Thread recommended** (may exceed 2000 chars) |
| `summary` | 10 | any | ~2,800–4,000 chars | Thread required |
| `detailed` | 1 | any | ~700–1,000 chars | Single message |
| `detailed` | 3 | any | ~2,000–3,000 chars | Thread required |
| `detailed` | 5–10 | any | ~3,500–8,000 chars | Thread required |
| `raw` | any | 5 | ~2,000–3,500 chars | Thread required |
| `raw` | any | 20 | ~7,000–14,000 chars | Thread required (multiple messages if needed) |
| `raw` | any | 50 | ~18,000–35,000 chars | Thread required (many messages) |

## 2. Panel Results Rendering — Single Run

The bot posts the XML tool output and Claude synthesizes it into a readable Discord message. The Claude agent converts the `<panel-results>` XML into natural-language Discord markdown. The following shows the natural-language equivalent the agent should produce.

### 2.1 Summary Format (3 dimensions) — Example Discord Message

```
**Panel Results: "Version A — emotional approach"** · Filipino Moms 30-45

**Stimulus** (social caption): *"Ang sarap, tipid pa! Lucky Me na naman kasama ko sa hapag-kainan."*

| Dimension | Mean | Mode | Distribution |
|-----------|------|------|-------------|
| Purchase Intent | 3.85/5 | 4 | ▁▂▅████▄ (1:1 2:2 3:4 4:9 5:4) |
| Message Clarity | 4.40/5 | 5 | ▁▁▂████▉▉ (1:0 2:1 3:2 4:8 5:9) |
| Personal Relevance | 4.10/5 | 4 | ▁▂▅████▅▅ (1:0 2:2 3:4 4:9 5:5) |

**Highlights**
> *"I'd definitely grab another pack — we always have Lucky Me at home."* — Maria Santos ✅
> *"I don't buy Lucky Me anymore. I switched to a cheaper brand."* — Celine Vera ❌

20 personas scored · Run ID: `f9e1c2d3`
```

**Notes on the above format**:
- `▁▂▅████▄` distribution bars are Unicode block characters — Claude generates these from the distribution data to give a visual sense of skew at a glance.
- The actual bot output will be generated by Claude interpreting the XML — the exact markdown format above is a target, not enforced output.
- Run ID is shown truncated to 8 chars for readability; the full ID is in the hint for copy-paste.
- Highlights show ✅ for positive valence, ❌ for negative.

### 2.2 Detailed Format — Thread Delivery

For `response_format="detailed"` with 3+ dimensions, the bot should:
1. Post a short acknowledgment message in the channel: *"Results are detailed — posting in a thread."*
2. Create or use an existing thread for this panel
3. Post full results in the thread (multiple messages if needed)

Each dimension's detailed output in a thread:

```
**Purchase Intent** · Mean: 3.85/5 · Std: 0.91 · Mode: 4 · 95% CI: [3.43–4.27]
Distribution: 1▌1 2▌▌2 3▌▌▌▌4 4▌▌▌▌▌▌▌▌▌9 5▌▌▌▌4

Positives:
> *"I'd definitely grab another pack — we always have Lucky Me at home."* — Maria Santos
> *"I'd pick this up on my next grocery run."* — Tess Aguilar

Negatives:
> *"I don't buy Lucky Me anymore. I switched to a cheaper brand."* — Celine Vera
> *"My doctor said to cut down on instant noodles."* — Rosa Dela Paz

Neutral:
> *"I buy it sometimes when I'm in a hurry, not really something I choose over fresh food."* — Ana Bautista
```

### 2.3 Raw Format — Thread Delivery, Multiple Messages

Raw format with 20+ personas will exceed 2000 chars per message. The bot should:
1. Post aggregated summary first (always fits in ~1 message)
2. Then post persona-by-persona breakdown across multiple messages, batched by ~1800 chars
3. Begin each batch with a continuation header: *"[Persona responses 1–7 of 20]"*, *"[8–14 of 20]"*, etc.

## 3. Comparison Results Rendering

### 3.1 Summary Comparison — Example Discord Message

```
**Panel Comparison: A vs B**

**A:** "Version A — emotional approach" · social caption
**B:** "Version B — practical approach" · social caption

| Dimension | A Mean | B Mean | Delta |
|-----------|--------|--------|-------|
| Purchase Intent | 3.85 | 4.12 | +0.27 ↑ ✨ |
| Message Clarity | 4.40 | 4.55 | +0.15 ↑ |
| Personal Relevance | 4.10 | 3.88 | -0.22 ↓ |

✨ = statistically notable (non-overlapping 95% confidence intervals)

**Bottom line:** Version B outperforms on purchase intent and clarity, but Version A resonates more personally with this panel. For driving purchase, Version B wins.

20 personas each · Run A: `f9e1c2d3` · Run B: `a4b5c6d7`
```

**Notes**:
- `✨` marks significant differences (non-overlapping CIs) — this is the Discord-rendered version of the `*` marker in the XML.
- The "Bottom line" synthesis is generated by Claude from the XML comparison data — the agent should summarize the comparison in 1–2 sentences.
- Both run IDs are shown (truncated) for reference.

## 4. Character Budget for Summary Format

To ensure `summary` format fits in a single Discord message (2000 char limit), the following character budget applies:

| Component | Max Characters |
|-----------|---------------|
| Header (panel name, run label, stimulus preview) | ~200 chars |
| Table row per dimension (name, mean, mode, dist) | ~120 chars each |
| 2 highlight quotes × 200 chars each per dimension | ~400 chars per dimension |
| Footer (personas scored, run ID) | ~60 chars |
| **Budget for 3 dimensions** | ~200 + 3×120 + 3×400 + 60 = **1,820 chars** |
| **Budget for 5 dimensions** | ~200 + 5×120 + 5×400 + 60 = **2,860 chars** (thread needed) |

**Conclusion**: Summary format is safe for up to 4 dimensions in a single message. For 5+ dimensions, the bot should post in a thread. This threshold applies regardless of panel size (quotes are truncated to 200 chars).

## 5. Progress Indicators (stub — full spec in w3-discord-ux)

> Note: The following is a stub for the w3-discord-ux aspect to expand. It establishes the pattern only.

When a user triggers `ssr_panel_run`:
1. Bot immediately replies: *"Running panel — this takes 3–8 seconds..."*
2. On completion: Bot edits or follows up with the full results.

When `ssr_panel_create` is called:
1. Bot immediately replies: *"Creating panel of {N} personas..."*
2. On completion: Bot follows up with the persona list summary.

For long `raw` format retrievals:
1. Bot replies: *"Fetching raw results — posting in thread..."*
2. Posts each batch to the thread.

## 6. Error Message Rendering

Tool errors from `ssr_panel_results` appear in Discord as Claude's natural-language interpretation. Examples:

| ToolError text | Expected Discord rendering |
|---------------|---------------------------|
| `"Run 'abc-123' not found. Use ssr_panel_list to see your available panels and their runs."` | *"I couldn't find that run. Use `/ssr_panel_list` or ask me to list your panels to find the right run ID."* |
| `"Run 'abc-123' is still in progress. Wait a few seconds and try again."` | *"That run is still being scored — give it a few seconds and I'll try again."* |
| `"No completed runs found for panel 'xyz-456'. Use ssr_panel_run..."` | *"That panel hasn't been run yet. Want me to run it now? What's the marketing asset you want to test?"* |
