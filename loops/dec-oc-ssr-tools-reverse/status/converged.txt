## What Was Built

A complete, implementation-ready specification for adding SSR (Semantic
Similarity Rating) consumer panel tools to Daimon — the decision-orchestrator
Discord bot. The specification lives entirely in tool-spec/ and enables a
developer to implement these tools by reading that directory alone.

## Tools Specified

1. ssr_panel_create  — Create a synthetic consumer panel with N personas
2. ssr_panel_run     — Run a marketing asset through a panel (SSR pipeline)
3. ssr_panel_results — Retrieve and compare run results
4. ssr_panel_list    — List panels owned by a Discord user
5. ssr_panel_delete  — Delete a panel and all associated data

## Architecture Summary

The SSR pipeline has 4 stages per persona:
  1. Stimulus presentation — persona system prompt + marketing asset prompt
  2. Text elicitation     — free-text reaction (no numeric ratings)
  3. Embedding            — text-embedding-3-small (1536 dims, $0.000006/persona)
  4. Anchor scoring       — cosine similarity → softmax-weighted Likert score

Aggregation across N personas yields: mean, std, mode, 95% CI, distribution
histogram, and qualitative highlights.

## Key Design Decisions

- Model: claude-haiku-4-5-20251001 for persona generation and response elicitation
- Embedding: text-embedding-3-small (already in pyproject.toml, no new dep)
- Concurrency: asyncio.gather across all personas per pipeline stage
- Scale: 5-point Likert (not 7) — cleaner semantics, better SSR reliability
- Personas: eagerly generated at panel_create time, stored in ssr_persona table
- Stimulus storage: inline in ssr_run (no separate ssr_stimulus table)
- Ownership: discord_id on ssr_panel, service-role only (no RLS needed)
- Platform: Platform.SSR = "ssr" added to platforms.py
